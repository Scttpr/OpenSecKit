NAME
    Quick Start - Sécurisation rapide

OBJECTIF
    Audit de sécurité et corrections de vulnérabilités


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. APPROCHE 1 : AVEC CLAUDE CODE (5 MINUTES) - RECOMMANDÉ

   1.1. Installer

      $ git clone https://github.com/Scttpr/OpenSecKit
      $ cd OpenSecKit/cli
      $ cargo install --path .

   1.2. Initialiser

      $ cd mon-projet/
      $ osk init

   1.3. Analyser une fonctionnalité

      $ claude
      >>> /security "système de login"

   1.4. Voir l'état global

      >>> /dashboard

   1.5. Corriger les risques critiques

      Claude Code affiche les vulnérabilités. Demandez-lui de les corriger :

      >>> Corrige RISK-LOGIN-001 (injection SQL dans src/auth/login.js:42)

   1.6. Vérifier la conformité

      >>> /audit

   Résultat : Analyse constitutionnelle + conformité aux 7 principes en 5 minutes.

   1.7. Structure des fichiers générés

      docs/
      ├── context/
      │   └── meta.md                  # Contexte projet
      └── security/
          ├── features/                # Analyses par feature
          │   └── SEC-*.md
          ├── risks/                   # Registre des risques
          │   ├── risk-register.yaml
          │   └── RISK-REGISTER.md
          ├── audits/                  # Rapports d'audit
          │   └── AUDIT-*.md
          ├── rgs/                     # Documents RGS
          │   ├── EBIOS-RM-*.md
          │   └── DOSSIER-HOMOLOGATION-*.md
          ├── continuity/              # Plans PCA/PRA
          │   ├── PCA-*.md
          │   └── PRA-*.md
          ├── incidents/               # Rapports d'incidents
          │   └── INC-*.md
          └── DASHBOARD.md             # Tableau de bord

2. APPROCHE 2 : MANUELLE (15 MINUTES)

   2.1. Modélisation menaces (5 min)

      $ cp templates/01-threat-modeling/stride-threat-model-template-planning.md \
           docs/security/threat-model.md

      Identifiez :
         - Actifs critiques (DB, secrets, sessions)
         - Top 3 menaces STRIDE
         - Contre-mesures

   2.2. Analyse risques (3 min)

      $ cp templates/02-risk-analysis/risk-scoring-template-planning.md \
           docs/security/risk-analysis.md

      Scorez chaque risque : Criticité × Probabilité × Exposition

   2.3. Actions immédiates (7 min)

      Sécuriser la DB : Requêtes préparées

      Installer détection secrets :

         $ brew install gitleaks
         $ gitleaks detect --source . --verbose

      Hook pre-commit :

         $ cat > .git/hooks/pre-commit << 'EOF'
         #!/bin/sh
         gitleaks protect --staged --verbose
         EOF
         $ chmod +x .git/hooks/pre-commit

      Planifier MFA : Ajouter dans le backlog
